# SIMPLEC 物理核查报告

## 背景

当前速度云图的主要问题已经从入口低速带转移到近壁扩散：入口速度已经能通过 `simple_enforce_inlet_velocity` 保持在来流速度附近，但汽车壁面附近的速度变化仍主要集中在第一层单元，流场内部看起来像“速度没有充分迭代/扩散”。

本报告记录当前 SIMPLEC 计算链路中每一部分的物理意义、已加入的运行时核查输出，以及当前代码压力/速度云图仍不够精确的更可能原因。

## 当前 SIMPLEC 计算流程的物理意义

1. **湍流黏性更新**
   - 每步由 `mu_t = rho*k/omega` 更新湍流黏性，并限制最大值。
   - 壁面相邻单元额外施加混合长度型涡黏性下限，只在 `mu_t` 不足时提高近壁扩散能力，不再强行压低 `k/omega/mu_t`。
   - 目的：让 no-slip 速度亏损通过内部面扩散到第二/第三层单元，而不是只在壁面面通量处形成一圈薄层。

2. **动量方程**
   - 使用压力梯度、对流、扩散和边界项组装速度方程。
   - 当前对角项包含伪瞬态稳定化；如果伪瞬态对角过强，会让速度场过度保持旧值，表现为速度云图接近初值，只在壁面第一层有变化。
   - 因此本次把 `SIMPLEC_PSEUDO_CFL` 从过小的 `0.05` 放宽到 `5.0`，并把 `SIMPLE_ALPHA_U` 从 `0.02` 提高到 `0.20`，降低“冻结速度场”的倾向。

3. **压力修正 / Rhie-Chow 通量**
   - 压力修正负责让速度满足连续性，并用 Rhie-Chow 通量抑制压力-速度棋盘振荡。
   - 输出中的 `pPrimeMax`、`pRange`、`aPuMin/aPuMax` 用于判断压力修正是否过强或动量对角是否过大。
   - 如果 `aP_u` 很大而速度变化很小，通常说明稳定化过强，速度扩散被对角项压制。

4. **入口速度约束**
   - 入口及其上游缓冲层保持固定来流速度，避免压力修正在入口内侧生成非物理低速带。
   - 运行时 `bcCheck inletUmin/inletUavg` 用于核查该问题是否复发。

5. **出口与质量守恒**
   - `massImb = inletFlux + outletFlux` 用于检查入口/出口质量通量是否平衡。
   - 若质量不平衡，速度云图异常可能来自压力修正矩阵或出口通量更新，而不是近壁模型。

## 新增运行时诊断说明

每个诊断步会额外输出三类核查：

```text
check iter massImb=... pPrimeMax=... pRange=... aPuMin=... aPuMax=... Uref=...
wallLayer iter U1=... U2=... U3=... Ufar=... muT1/mu=... muT2/mu=...
bcCheck iter inletUmin=... inletUavg=... outletUavg=... fluxSum=...
```

含义如下：

- `massImb`：入口/出口质量通量和，应该接近 0。
- `pPrimeMax`：当前压力修正幅值，过大说明压力方程或松弛可能不稳。
- `aPuMin/aPuMax`：SIMPLEC 速度修正分母。过大时速度场会被伪瞬态/对角项冻结。
- `wallLayer U1/U2/U3/Ufar`：按壁距分层的平均速度，用于直接判断速度亏损是否只停留在第一层。
- `muT1/mu`、`muT2/mu`：近壁层涡黏性相对分子黏性的倍数，用于判断近壁扩散能力是否不足。
- `inletUmin/inletUavg`：入口速度是否仍为来流速度。
- `fluxSum`：入口/出口通量是否守恒。

## 当前更可能的误差来源

1. **此前伪瞬态对角过强**
   - 过小的 `SIMPLEC_PSEUDO_CFL=0.05` 会生成很大的动量对角项。
   - 这会让速度场强烈保持旧值，近壁变化难以传入流场内部。
   - 本次已改为 `SIMPLEC_PSEUDO_CFL=5.0`，降低冻结效应。

2. **速度欠松弛过强**
   - `SIMPLE_ALPHA_U=0.02` 会让每步动量更新非常小。
   - 在粗网格和强伪瞬态对角共同作用下，云图容易像“没有迭代过”。
   - 本次已改为 `SIMPLE_ALPHA_U=0.20`。

3. **近壁涡黏性不足或只作用于壁面相邻单元**
   - 若 `wallLayer` 输出显示 `U1` 明显变化但 `U2/U3` 接近自由流，则近壁扩散仍不足。
   - 当前用混合长度型 `mu_t` 下限提升第一层向内部面的扩散能力。

4. **网格壁距分层不足**
   - 如果 `wallLayer` 中只有 `U1` 有计数，`U2/U3` 为 0，说明当前测试网格本身壁距分层太粗或 `wallDistance` 对所有单元几乎相同。
   - 这时云图“只在第一层变化”可能部分来自网格分辨率/壁距计算，而不仅是求解器。

5. **压力-速度耦合仍需和真实汽车网格共同验证**
   - smoke mesh 只有 24 个单元，无法可靠代表汽车外流近壁层传播。
   - 对真实汽车网格，建议重点查看 `wallLayer` 输出中 `U1/U2/U3` 是否呈连续过渡，以及 `muT1/mu` 是否不再过低。

## 建议的判据

- 入口问题已解决的判据：`bcCheck inletUmin` 与 `inletUavg` 接近来流速度。
- 质量守恒判据：`abs(fluxSum)` 接近 0。
- 近壁扩散改善判据：`wallLayer U1/U2/U3` 不应只有 `U1` 变化，`U2` 和 `U3` 应该出现连续速度亏损。
- 若 `wallLayer U2/U3` 始终为 0，优先检查 `wallDistance` 计算或网格壁法向分辨率。
