# SIMPLEC 方法与调试简要报告

> 注：这里的 “SIMPLIC” 按当前代码和文档中的 SIMPLEC 压力基求解器理解。

## 1. 当前 SIMPLEC 采用的主要方法

1. **压力基低速不可压/弱可压求解框架**
   - 当前 `SOLVER_SIMPLE` 路径使用 SIMPLEC 类型的压力-速度耦合：先解动量方程预测速度，再解压力修正方程修正速度、压力和面质量通量。
   - 压力以 gauge pressure 方式参与修正，密度使用参考密度，适合当前低 Mach 外流算例。

2. **完整三维压力-速度耦合**
   - 动量方程、Rhie-Chow 面通量和速度修正都使用三维压力梯度，而不是只沿 x 方向修正。
   - 这样做的目标是避免压力/速度云图退化成近似一维分布，并让车身上下、侧向和尾部区域都能通过压力修正参与耦合。

3. **Rhie-Chow 面通量插值**
   - 面质量通量使用 Rhie-Chow 修正来抑制同位网格上的压力-速度棋盘振荡。
   - Rhie-Chow 中的速度修正系数与动量方程中的 `aP_u` 保持一致，并加入阻尼参数，使通量修正不过度放大。

4. **SIMPLEC 型动量对角与压力修正方程**
   - 动量方程保存邻居系数和，并用 `aP - H` 形式得到 SIMPLEC 速度修正分母 `aP_u`。
   - 压力修正方程使用 PCG 迭代求解，而不是简单点迭代，以改善压力修正的收敛和稳定性。

5. **方程层稳定化而非全场硬裁剪**
   - 当前主要通过动量欠松弛、压力欠松弛、伪瞬态对角项、压力步长限制和 Rhie-Chow 阻尼稳定计算。
   - 这类方法作用在方程和修正量层面，目标是避免用全场速度/压力硬裁剪掩盖真实的耦合问题。

6. **RANS k-omega 与近壁扩散增强**
   - 湍流黏性按 `mu_t = rho*k/omega` 更新，并对近壁单元加入混合长度型涡黏性下限和上限。
   - 粗网格壁面附近还可通过 wall diffusion boost 增强壁面相邻面的扩散，使 no-slip 影响不只停留在第一层单元。

7. **OpenFOAM 边界名映射与边界条件处理**
   - 代码读取 OpenFOAM `boundary` 中的边界名，并将 `inlet`、`outlet`、`wall/airfoil`、`symmetry/empty` 等映射到内部边界类型。
   - SIMPLEC 中入口固定来流速度，出口进行压力/回流稳定处理，壁面为 no-slip，symmetry/empty 近似按 slip 零法向通量处理。

## 2. 已做的主要调试方向

1. **修复一维化压力耦合问题**
   - 对比发现 SIMPLEC 中若只用流向压力梯度，压力和速度容易呈现近似一维分布。
   - 已将动量源项、Rhie-Chow 预测通量和速度修正统一为完整三维压力梯度。

2. **降低速度场“冻结”倾向**
   - 早期设置中过强的伪瞬态对角和过小的速度松弛会使速度场几乎保持初值，只在壁面第一层变化。
   - 已放宽 `SIMPLEC_PSEUDO_CFL`，提高 `SIMPLE_ALPHA_U`，并通过 `aP_u` 诊断观察动量对角是否过大。

3. **替换压力修正求解方式**
   - 压力修正从点 Gauss-Seidel 式迭代改为对角预条件 PCG。
   - 同时加入自适应压力步长限制，避免单步压力修正过大导致压力场振荡或发散。

4. **入口低速带调试**
   - 增加入口及入口缓冲区速度保持逻辑，避免压力修正在入口附近形成非物理低速带。
   - 通过 `bcCheck inletUmin/inletUavg` 输出确认入口速度是否接近来流速度。

5. **近壁扩散调试**
   - 针对“车身壁面附近速度只在第一层变化”的问题，加入近壁 `mu_t` 下限和壁面扩散增强。
   - 新增 `wallLayer U1/U2/U3/Ufar` 与 `muT1/mu`、`muT2/mu` 输出，用于判断速度亏损是否向第二、第三层传播。

6. **出口回流与质量守恒调试**
   - 对出口回流加入有界已知值源项，减少回流导致的动量方程不稳定。
   - 增加入口/出口通量、`massImb`、`fluxSum` 等诊断，直接检查连续性是否满足。

7. **边界条件读取调试**
   - 增加边界名读取和映射打印，避免 OpenFOAM 边界顺序变化导致入口、出口、壁面或 symmetry 被误识别。
   - 对 symmetry/empty 边界按 slip/零法向通量处理，避免把对称面误当作 no-slip wall。

8. **自动化运行检查**
   - 增加 `check_simplec_350.py`，用于运行到指定 SIMPLEC 迭代步并解析速度、压力差和质量通量等范围。
   - 该脚本不是完整物理验证，但可以快速发现 NaN、明显发散、入口速度失真或压力范围异常。

## 3. 当前仍需重点观察的现象

- **近壁速度亏损是否进入第二/第三层单元**：如果 `wallLayer U2/U3` 长期接近自由流，说明近壁扩散或网格壁距分层仍不足。
- **压力极值位置是否合理**：如果压力极值长期只在入口/出口而车身表面差异很弱，应继续检查压力修正边界和 Rhie-Chow 通量一致性。
- **质量通量是否平衡**：`massImb` 和 `fluxSum` 应接近 0，否则速度云图问题可能来自压力修正矩阵或出口边界。
- **调试网格代表性**：小 smoke mesh 只能检查程序不崩溃；车身外流的壁面传播和尾迹压力仍需要真实汽车网格验证。

## 4. 简要结论

当前 SIMPLEC 的核心思路是：用压力基 SIMPLEC 框架求低速外流，用三维压力梯度和 Rhie-Chow 通量保证压力-速度耦合，用 PCG 压力修正和方程层阻尼提高稳定性，并通过近壁湍流扩散、入口保持、出口回流处理和运行时诊断逐步定位速度云图与压力云图异常。已经完成的调试主要集中在三维耦合恢复、速度场冻结、入口低速带、近壁扩散不足、出口回流、质量守恒和边界名映射几个方面。
